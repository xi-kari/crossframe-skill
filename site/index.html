CrossFrame Skill Suite

给 AI 装上的中文结构思考系统。

不是让 AI 更快回答，而是让 AI 在回答前先交账，在回答后还能继续追问。

CrossFrame 会先拆清事实、证据、尺度、机制、判断档位和行动边界，再生成可审查、可追问的中文输出。

这是项目介绍页；CrossFrame 作为本地 AI skill 使用，推荐安装到 Codex、Claude Code 或其它支持项目规则的 agent。

How It Works

CrossFrame 让 AI 先做四件事

1

拆事实

事实、解释、推测和来源分开，不让顺滑表达替代证据。

2

定尺度

个人、关系、组织、制度和历史阶段不能混写成同一种判断。

3

交台账

中心命题必须绑定 claim_id，来源必须绑定 source_id。

4

过审查

正文不能强于台账，生成层不能自己宣布质量通过。

Use Boundary

什么时候适合用 CrossFrame？

适合
  • 复杂问题，需要拆事实、机制和边界。
  • 写文章前，需要先形成结构底稿。
  • AI 输出看起来顺，但你想检查它有没有真正推理。
  • 分析结束后，还想继续追问、反证和迁移。
不适合
  • 单纯事实查询。
  • 普通改写、翻译、摘要。
  • 没有要求 CrossFrame 的日常聊天。
  • 需要医生、律师、财务顾问或正式调查程序的高责任结论。

    FAQ

    常见问题

    它会自动接管所有问题吗？

    不会。CrossFrame 是 explicit-only，只有用户明确点名 CrossFrame 或对应 skill 时才启动。

    它和提示词模板有什么不同？

    提示词模板通常要求“说得更好”。CrossFrame 要求回答前交台账，回答后可审查、可追问、可降档。

    为什么要有 crossframe-inquiry？

    因为完整分析、成文和 review 结束后，真正有价值的下一步通常是反证、补证、迁移和行动边界确认。

    这个网页能直接运行 CrossFrame 吗？

    不能。这个页面是介绍页。CrossFrame 作为本地 AI skill 使用，需要安装到 Codex、Claude Code 或其它支持项目规则 / skills 的 agent 中。

    为什么首页示例都是虚构或匿名的？

    因为 CrossFrame 会处理历史、公共、组织等高责任主题。首页只展示工作方式，不对现实对象做判断；真实材料需要进入 source ledger、claim ledger 和 review。

    它会让 AI 变慢吗？

    复杂任务会更慢，因为它要先拆事实、证据、命题和边界。CrossFrame 的目标不是最快回答，而是降低顺滑错误和越界判断。
