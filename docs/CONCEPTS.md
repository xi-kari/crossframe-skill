# 核心概念

## Claim Ledger

`claim ledger` 是命题台账。它要求中心命题、机制句、行动建议、公共定性、文章转译和高风险概念判断都有稳定 `claim_id`。

没有 `claim_id` 的强判断不得进入正文或 review 通过项。

## Source Ledger

来源台账使用十字段口径，核心是：

- `source_id`
- 来源
- 时间
- 来源类型
- 支持的 `claim_id` / 命题
- 不能证明什么
- 证据档位
- 使用位置
- 降档理由
- 仍需补证处

来源不能只“看起来相关”，必须能说明它支持哪个 `claim_id`，也必须说明它不能证明什么。

## Concept Contract

概念契约用于约束高风险概念。比如责任链、开放断言、权力封闭、低条件行动、退出转移、修复副产品等概念，不能只按字面使用。

使用这些概念前，需要读对应概念卡，并确认现实证据、失败机制、行动上限和撤回条件。

## Read-State Capsule

`v5-read-state-capsule` 记录本轮读过哪些 source modules、连续联读包和源锚点。它不是正文，也不是自我盖章。

如果中心命题、机制候选、高风险概念或行动边界不能回指胶囊源锚点，必须删除、降档，或标为“本文推断 / 表达转译 / 外部思想映射”。

## Review 与 Inquiry

`crossframe-review` 是质量闸。它负责检查输出是否真正推理、是否守证据边界、正文是否强于台账、生成层是否自我盖章。

`crossframe-inquiry` 是完成态后的追问层。它复用上一轮 `claim ledger`、机制候选、概念契约、文章正文和 review 结果，只生成追问、反证、补证、迁移和行动边界，不重新发明新结论。
