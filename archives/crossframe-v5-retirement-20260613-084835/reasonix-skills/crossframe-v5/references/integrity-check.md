# v5 完整性检查

本文件是 CrossFrame v5 输出前总闸，合并连读包检查、概念保真、七闸、低权力保护、AI/过程产物降级和退场规则。

## 五条铁律

1. **连读包先于概念卡。** 概念卡不能单独承担强判断。
2. **七闸先于内容。** G1-G7 没过，不能进入实质分析、公开判断或行动建议。
3. **源锚点先于框架推断。** 不能回指 v5 源结构锚点的内容，不得写成 CrossFrame v5 原义。
4. **胶囊先于重复读源。** `crossframe-v5` 已生成 `v5-read-state-capsule` 时，下游默认不得重复整块读取源索引或完整连读包。
5. **装配先于收尾。** 默认长文链路的最终可见输出必须保留底稿和正文，review 只能作为总闸附录。

| 检查项 | 必须回答 | 失败处理 |
| --- | --- | --- |
| Source modules | 本次判断命中了哪些 v5 source modules？是否先由模块推出连读包？ | 缺失则回到 `v5-material-selection-map.md` 定向补读；不得直接跳单卡。 |
| 连读包 | 本次判断触发了哪些 v5 连读包？是否按 `v5-continuity-bundles.md` 完成同读？ | 补读；来不及补读则降档。 |
| 读态胶囊 | 是否有 `v5-read-state-capsule`？是否列出触发理由、源锚点、相邻约束和降档规则？ | 缺失则回到 `crossframe-v5` 生成；不得让下游各自重读全量源。 |
| 源锚点 | 中心命题、机制候选、关键概念和行动边界能否回指胶囊源锚点？ | 无锚点则删除、标为本文推断/思想映射，或降档。 |
| 七闸 | G1-G7 是否全部通过？跳闸是否标注理由和补偿措施？ | 不得进入实质判断或行动建议。 |
| 低权力主体 | 谁可能因判断受损？是否有安全提交反例、代理提交、反报复和撤回路径？ | 先保护处境，暂停强判断。 |
| 概念保真 | 承担判断作用的概念是否读了所属连读包和相邻概念？ | 只读单卡即输出判为不合格。 |
| AI/过程产物 | 证据是否来自 AI 文本、表格流程、内部报告或平台指标？ | 降级为线索，不得作为现实证明。 |
| 观测影响 | 诊断、命名、评分或公开是否改变对象？是否区分基线/观测/发布后反应？ | 加入观测影响登记；必要时降档。 |
| 强判断程序 | 是否具备反向条件、证据要求、申诉入口、反报复、证据补充、外部复核、回滚和版本写回？ | 缺任一项不得公开强判断。 |
| 领域词 | 强领域词是否翻译成结构变量？是否公开规范性前提？ | 不得用领域词直接证明结论。 |
| 退场 | 本场景是否更适合外部框架、专业程序、现实调查或安全转接？ | 声明边界并转接/降级。 |
| 自动链路 | suite 默认链路是否从选择一路执行到 essay 和 review？ | 若停在短推理提纲或正文后要求“继续”，判链路失败。 |
| 最终装配 | 默认长文链路是否保留 `结构洞察底稿`、`文章正文` 和 `CrossFrame v5 Review`？ | 若最终只输出 review，判装配失败；补回上游底稿和正文后再交付。 |

## 孤立读取硬失败

以下情况直接判为不合格：

- 只读“爱/开放行动”就给亲密关系或牺牲判断。
- 只读“开放断言”就影响资源、名誉、资格、权利或公共记忆。
- 只读“七闸”就跳过源结构和场景包。
- 只读“AI 合规/恶意合规”就判断机构现实有效。
- 只读“领域词翻译”就把阶级、资本、国家等写成框架本体结论。
- 只读“框架治理”就给 CrossFrame v5 自证安全。
- 无 `v5-read-state-capsule`，却让 essay/review 各自重复整块读取源索引。
- 无 v5 source modules，直接从请求跳到连读包或 concept card。
- 中心命题、机制候选或行动边界没有源锚点，却写成 v5 框架结论。
- suite 默认长文链路停在短推理提纲、文章正文或任一中间产物，要求用户说“继续”。
- suite 默认长文链路已经生成底稿和正文，但最终只输出 `CrossFrame v5 Review`。

## 显示规则

普通输出只显示短行：

```text
- 本次 v5 连读包：...
```

完整工作表仅在用户要求审计、复盘或 skill 设计时展开。
