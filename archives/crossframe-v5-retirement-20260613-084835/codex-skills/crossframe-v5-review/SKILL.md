# CrossFrame v5 Review


> **本 skill 不独立触发。** 所有 CrossFrame v5 任务统一从 `crossframe-v5-suite` 入口调度。用户无需直接调用本 skill；suite 根据路由规则在需要时自动加载。

如果评审对象来自多个 CrossFrame v5 skill 的连续工作流，先读取 `../crossframe-v5-suite/SKILL.md` 还原应有调度链，再判断是否有漏触发、误触发或跳过质量闸。

本 skill 只做评审与修复建议，不替代 `crossframe-v5` 生成诊断，也不替代 `crossframe-v5-essay` 写文章。中文为权威语义；英文只作 skill id、文件名和接口说明。

## 装配边界

当本 skill 作为 suite 默认长文链路的最后一环时，review 只产生 `review_result`，不得覆盖上游的 `结构洞察底稿` 和 `文章正文`。最终可见输出必须按以下顺序装配：

```text
# 结构洞察底稿
# 文章正文
# CrossFrame v5 Review
```

只有用户明确要求“只要评审 / 只评已有文章 / 不要正文”时，才允许只输出评审报告。否则，默认长文链路最终只输出 `CrossFrame v5 Review`，即使评审本身正确，也判为装配失败。

## 轻入口规则

每次触发后，先读取或接收 canonical 状态，而不是复制它们的正文：

1. 接收 suite 传入的 `selection_state`，以及 `crossframe-v5` 传入的 `v5-read-state-capsule`、短推理提纲、文章底稿和正文。
2. 读取 `../crossframe-v5/references/integrity-check.md`，判断本应触发哪些 v5.0 连续联读包。
3. 若胶囊缺失，读取 `../crossframe-v5/SKILL.md` 与 `../crossframe-v5/references/read-routing-map.md` 还原路线。
4. 若评审对象是深度、审计、高责任、公共制度、亲密关系、长期演化或文章类输出，优先检查 `v5-read-state-capsule` 与 `../crossframe-v5/worksheets/source-continuity-check.md`；只有胶囊不足时才按锚点定向查阅 `../crossframe-v5/references/v5-continuity-bundles.md`。
5. 若评审对象是文章、长文、评论、思想文章、报刊答复或“现代编辑同志口吻”输出，按需读取 `../crossframe-v5-essay/SKILL.md`。
6. 读取本目录的 `protocols/review-protocol.md` 和 `templates/review-report.md`。
7. 若涉及文章底稿、引用、检索材料或声口，追加读取 `protocols/article-review-protocol.md`。
8. 按需读取本目录 `references/` 中的评分表、失败类型表和证据边界清单。

不要把 CrossFrame v5 主 skill 或文章 skill 的完整内容复制到本 skill 输出中。评审时只引用必要规则名、触发点和证据位置。若 `crossframe-v5` 已提供 `v5-read-state-capsule`，不得重复整块读取源索引；只检查胶囊是否足以支撑输出。

## 评审目标

判断一个输出是否真的完成了 CrossFrame v5 的最低推理链：

- 明确诊断或写作对象。
- 区分事实、解释、证据缺口和判断档位。
- 经过七闸：有效对象、证据追踪、受影响位置、权力与反报复、中介与反身性、强判断程序、行动上限。
- 至少形成两个机制候选，或说明为什么只能有一个。
- 对承担判断作用的高风险概念做保真检查，而不是把术语当结论。
- 对属于 v5 连续板块的高风险概念做 source module 与源结构连续性检查，而不是只读单张概念卡。
- 对中心命题、机制候选、概念上升和行动边界做源锚点校验，而不是把模板词当作源文本依据。
- 给出可撤回条件、下一步观察或低条件行动边界。
- 文章类输出必须先有结构洞察底稿，再写正文。
- suite 默认链路必须自动执行到 review；若前序输出要求用户说“继续”才到本 skill，判为链路失败。
- suite 默认长文链路必须在 review 后保留上游底稿和正文；若最终只剩评审报告，判为装配失败。

## 必抓失败

以下问题一旦出现，要在评审中明确定位；严重时直接判为不合格：

- 概念堆砌：只堆“承接、回流、尺度、反俘获”等词，没有落回事实和行为。
- 伪推理：先给结论，再用术语装饰；没有机制候选、反向条件或证据闸。
- 事实边界缺失：把传闻、AI 报告、自评、搜索摘要或解释当事实。
- 跳过底稿：文章类输出直接成文，没有结构洞察底稿或等价内部骨架。
- 人格审判：把结构诊断变成“这个人坏、懒、无能、病态”等定性。
- 伪造引用：编造原文、页码、出处、作者观点；不确定原句却写成直接引用。
- 查源接管命题：检索材料决定文章立场，CrossFrame v5 只变成资料拼贴外壳。
- 强判断越级：处分、名誉、权利、资源、公共记忆类判断没有命题验证和申诉/反证入口。
- 尺度洗白：用宏观叙事取消低尺度痛苦、责任链或具体失职。
- AI 合规剧场：把 AI 生成材料、漂亮报告或自评文本当作独立强证据。
- 连续性保真失败：本应触发 `v5-continuity-bundles.md` 的联读包，却只读单个概念卡、单个 protocol 或单个摘录就下判断。
- 源锚点失败：中心命题、机制候选、概念上升或行动边界无法回指 `v5-read-state-capsule`，却写成 CrossFrame v5 原义。
- 胶囊失效：`crossframe-v5` 未生成源结构摘要，导致 essay/review 反复整块吞入源索引或各自发明路由。
- 链路断裂：默认长文任务停在短推理提纲、底稿或正文后等待用户输入“继续”。
- 装配失败：默认长文链路已经生成底稿和正文，但最终可见输出只剩 review，导致用户拿不到文章交付物。
- v5 现实保护失败：涉及 AI 现实验证、弱信号、不透明、无制度基础设施、无法退出、恶意合规、隐喻漂移、工具化或开放断言退场，却没有读取对应 v5 概念卡和联读包。

## 输出协议

默认使用 `templates/review-report.md`。最终评审必须包含：

- 评审对象
- 事实边界
- 触发规则
- 评分/等级
- 关键问题
- 证据定位
- 修复建议
- 是否合格

若用户只要一句话结论，也要保留“是否合格 + 主要失败点 + 下一步修复”的最小结构。

若用户没有明确只要评审，且上游已生成文章底稿和正文，本 skill 的报告必须作为最终装配的第三部分出现；不得单独成为最终回答。

## 合格判定

评分只是辅助，硬失败优先：

- A：90-100，合格。
- B：75-89，条件合格，小修后可用。
- C：60-74，不合格，需要大修。
- D：40-59，不合格，需要重做主要推理。
- F：0-39 或触发硬失败，高风险失败。

触发人格审判、伪造引用、跳过文章底稿、强判断越级、证据边界完全缺失、连续性保真失败时，即使文字流畅，也不能判为合格。

## 修复原则

评审输出优先给可执行修复，不默认重写全文。除非用户要求“直接改写”，否则只给：

- 应补的事实边界。
- 应读取或声明的路由。
- 应新增的机制候选。
- 应降档的判断。
- 应删除或改写的人格审判、伪引用和术语堆砌句。
- 文章应补的结构洞察底稿项目。


## v5 连读要求

本专项 skill 不独立决定源结构。进入本 skill 前，必须已经由 `crossframe-v5` 根据 `../crossframe-v5/references/read-routing-map.md` 选定 v5 连读包；输出前必须回到 `../crossframe-v5/references/integrity-check.md` 做完整性检查。不得只读本 skill 或单张 concept card 就输出强判断。
