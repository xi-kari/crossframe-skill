# CrossFrame Max

`crossframe-max` 是 CrossFrame 的独立最大展开入口。它用于把一个对象、一件事、一段材料或一个问题当作局部世界来建模，尽当前材料、工具和上下文允许的最大努力，展开世界观理解、结构诊断、演化路径、处理问题、正文成文和续写索引。

## max-clean-runtime-entry

运行入口保持正向、精简：`SKILL.md` 只放运行 contract、读法地图、执行顺序和交付闸门。失败样本和反例压力测试放在 `evals/`；长细则放在 `protocols/`、`templates/` 和脚本。

## 触发边界

这是 explicit-only skill。只有用户明确要求 `crossframe-max`、最大尺度、穷尽推演、完整解释、无限制长文、全尺度世界观解释或类似意图时使用。

## 世界观层先行

先理解框架如何看世界，再进入事件。诊断层是世界观进入事件后的分析动作；疗愈、行动和成文属于应用层。

执行时必须把对象当作一个局部世界，也就是把一件事当作一个局部世界来对待：

- 先做局部世界建模，再做判断。
- 先识别运行规律，再定位问题。
- 先展开演化路径推演，再给处理问题路径。
- 先保留证据边界，再写完整长文。

## 资料前沿

涉及真实机构、公共事实、历史事件、法律政策、平台规则、技术标准、人物、公司或最新情况时，主动检索与反向检索并行。

`max-source-frontier` 必须登记：

- 支持材料、反对材料、冲突材料、缺失材料和不可访问材料。
- 未找到也要登记，标明已查未得、暂不可访问或仍需补证。
- 来源类型分层、资料快照时间、缺席主体检查、检索反身性和停止条件。
- 资料穷尽不等于现实穷尽，不能假装穷尽现实真相。

## 爱与超越性窗口

`max-transcendence-window` 只处理结构解释到达边界后仍然出现的开放行动可能性。

核心判断：

- 不能解释，不等于出于超越性。
- 开放行动能力要同时检查非工具性、非占有性、真实成本但不转化成债权、保留对方的自由、不取消边界。
- 不得要求受害者继续忍耐。
- 误读为爱时，要检查创伤重复、控制、补偿、角色依赖、道德表演和权力策略。

## 连续运行状态

`max-continuation-ledger` 必须保存：

- 已读材料、已展开路径、未展开路径、已撤回判断、已降档判断。
- 下一轮续写入口。
- 防重复、防漂移、术语漂移、主体漂移、判断强度漂移和路径遗漏检查。

## 反向推演

`max-red-team-pass` 必须至少回答：

- 如果这套解释是错的，最可能错在哪里？
- 哪些主体、路径、尺度、材料或沉默位置被框架偏好遮蔽？
- 是否为了完整解释制造解释力幻觉？
- 哪些反例会撤回、降档或改写中心命题与行动建议？

## 主体与路径

`max-position-matrix` 必须列出行动者、承接者、受害者、旁观者、制度主体、沉默者、退出者和未来主体。

`max-path-confidence-layers` 必须区分事实路径、机制候选路径、低置信想象实验、纯反事实路径和价值性解释路径。不能把所有可能写成同一种强度。

`max-unexhaustible-declaration` 必须保留内心动机、未来自由行动、沉默主体经验、未公开材料、历史偶然性和超越性窗口。穷尽一切从狂妄改成诚实。

## 模板与正文闸门

执行 `template-fidelity gate`：`templates/max-dossier-output.md` 和 `templates/max-essay-output.md` 是 contract。

执行 `longform-dominance gate`：`max-dossier` 保存结构底稿，`max-essay` 承担最终完整解释。

正文完整性按三档判断：

- 最低通过：`max-essay` 可见字符数至少达到 `max-dossier` 的 1.6 倍。
- 强完成：`max-essay` 至少达到 2.2 倍，并把主要路径写入连续解释。
- 最大完成：`max-essay` 至少达到 3.0 倍，或剩余内容只属于非阻断续写分支。

解释覆盖率必须检查：局部世界、运行规律、问题形成、路径演化、处理问题、伪修复、资料前沿、主体位置矩阵、路径置信分层、反向推演、超越性窗口、不可判断区、撤回条件、不可穷尽声明和续写索引是否进入正文。

交付前运行：

```bash
python scripts/check_crossframe_max_artifacts.py --workspace <artifact-dir>
```

在 skill 目录外运行时，优先使用当前 `crossframe-max` 随附脚本；在 canonical repo 中运行时，也可使用仓库根目录脚本。
