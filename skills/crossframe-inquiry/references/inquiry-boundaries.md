# Inquiry Boundaries / 追问边界

## 1. 追问不是诊断升级

追问产生的是问题，不是新结论。用户回答可能触发新材料，但新材料必须经过：

```text
inquiry_delta -> claim ledger delta -> crossframe / review
```

不得在 inquiry 中直接升级判断。

## 2. 追问不是心理审判

以下问法禁止默认使用：

```text
你为什么总是这样？
你是不是不敢面对？
你是不是也有责任？
你为什么不直接行动？
```

改为：你现在更想澄清事实、判断机制，还是先保护行动边界？

## 3. 追问不是行动催促

公共、组织、亲密关系、高责任主题中，追问必须先问：

- 现在是否有足够事实？
- 行动上限是什么？
- 停止条件是什么？
- 谁可能承担风险？

## 4. 追问不是概念考试

不得默认问“你能解释什么是承接/回流吗？”除非用户要求教学。更好的问法是：在这个案例里，谁承担了没有写回制度的成本？这个成本有没有变成规则、资源或边界的改变？

## 5. 追问不是无限循环

每轮追问最多 3-5 个问题。连续两轮后必须提供收束选项：

```text
A. 继续深挖
B. 收束成摘要
C. 转成行动边界
D. 回到原文重写
E. 进入 review
```

## 6. 发布边界

追问结果默认不用于发布。若用户要发布追问后的新判断，必须回到 `crossframe-review`。

## 7. 检索不是授权

`crossframe-inquiry` 可以检索 sibling skill 的协议、references 和 templates，但检索不是专项判断授权。

- 读了 `crossframe-org`，不等于已经完成组织诊断。
- 读了 `crossframe-public`，不等于已经完成公共事实核验。
- 读了 `crossframe-history`，不等于已经完成史料台账。
- 读了 `crossframe-debate`，不等于已经完成命题辩论。

检索材料只能帮助生成更好的问题和边界。若用户需要结论、方案、发布判断或正式审查，必须回到 suite 路由对应 sibling skill。
