# 输出模式与角色选择器

本文件定义 CrossFrame 分析的输出模式（推断风险偏好）和角色（目标受众与分析立场），作为正交于推理链的第二个维度。模式×角色的组合只改变前台表达风格和推断边界，不降低概念保真、机制候选、判断档位和连续联读的任何要求。

## 一、输出模式（4档）—— 决定"冒多大推断风险"

| 模式 | 触发词 | 内部行为 |
|------|-------|---------|
| 🔵 **保守** | "保守一点""只确认最稳的""稳一点" | 强制留两处边界/反例；开放断言以上不输出；所有判断写撤回条件；机制候选至少两个且每个必须有不成立的反向信号 |
| 🟢 **客观** | "客观""不站队""平衡""正反都看"（默认） | 正反方等权重；机制候选必须两个以上且都有支持证据；不输出单独的批评段落（批评融入正反对比）；撤回条件与支持条件对称 |
| 🟠 **激进** | "激进""最大胆的判断""直说" | 允许单一主机制（但必须注明"此为当前最稳假说，竞争机制见底稿"）；允许推断链延伸两层；边界段缩短为一句话（但仍必须存在）；撤回条件仍需写清 |
| 🔴 **批判** | "批判""找问题""质疑""反驳""有什么问题" | 以反例、弱信号、隐藏代价为主要组织方式；肯定性内容压缩至总体上不超过全文 20%；强制包含"最强反方"段（即使作者本人不同意）；指出盲点、利益偏向或副作用 |

## 二、角色选择（6种）—— 决定"为谁分析、用什么语言"

| 角色 | 触发词 | 后台推理引擎 | 前台表达风格 |
|------|-------|-------------|-------------|
| 🎓 **1. 学术专家** | "学术""严谨""引用理论""学者视角" | 诊断协议 + 概念解释协议 + 概念上升协议（必开） | 概念定义先于判断；保留条件与边界；可引用经典但只能意译/思想映射；结论写"当前证据支持……但若出现X则需修正" |
| 🔧 **2. 实践工匠** | "实操""怎么做""经验""落地" | 诊断协议 + 低条件行动协议 | 步骤/工具/周期/成本/常见坑点；用一线案例说明成败关键；结论写"先做A，观察B，如果C则停/改" |
| 🎯 **3. 战略决策者** | "战略""决策""优先级""投入产出" | 诊断协议 + 递进协议 + 治理连续性协议 | 目标-资源-风险-收益平衡；"要不要做、投入多少、何时止损"；全局优先级排序；保留退出条件和替代路径 |
| 📣 **4. 大众传播** | "通俗""易懂""给普通人看""简单说" | 诊断协议 + 表达翻译协议 | 比喻/故事/直观感受；避免术语（必须用的只做括号注释）；给可直接记住的建议；情绪共鸣但不煽动 |
| 🔍 **5. 批判反思者** | "批判反思""质疑前提""反方""盲点" | 诊断协议 + 命题验证协议 + 反俘获协议 | 质疑前提、寻找反例和潜在代价；指出盲点、利益偏向或副作用；对冲群体共识；强制包含"如果这个判断错了会怎样"段 |
| 🔮 **6. 未来探索者** | "未来""趋势""5年后""推演""演变" | 推演协议（`inference-protocol.md`）+ 生命周期协议（`lifecycle-diagnosis-protocol.md`）+ 表达翻译协议 | 基于边缘信号推演 5-10 年；讨论技术/社会/伦理非线性变化；多分支展开（至少两个分歧路径）；每个分支写触发条件和反向信号 |

### 未来探索者角色的内部路由（自动触发）

当角色 = 6（未来探索者）时，自动触发以下路由：

1. 读取 `../crossframe/protocols/inference-protocol.md`（推演协议——多分支展开、触发条件、反向信号、终点判断）
2. 读取 `../crossframe/protocols/lifecycle-diagnosis-protocol.md`（生命周期阶段判断——作为推演的诊断基础）
3. 读取 `../crossframe/protocols/expression-translation-protocol.md`（前台表达翻译——把后台推演链翻译成人话）
4. 输出模板使用 `../crossframe/templates/inference-output.md`（从底稿转译推演→前台故事化叙事）
5. 至少展开两个分歧路径，每个路径写清："触发条件是什么、可能的终点状态是什么、什么信号会让这个路径关闭"

### 各角色的默认声口

| 角色 | 默认声口 | 说明 |
|------|---------|------|
| 学术专家 | 中性分析体 | 不启用现代编辑底色；结论保留条件和边界 |
| 实践工匠 | 中性分析体（可带决定语气） | "建议先做X，观察Y" |
| 战略决策者 | 中性分析体（可带决定语气） | "优先级排序：A>B>C" |
| 大众传播 | 可启用编辑底色（答复体/评论体） | 通俗、亲切、有共鸣 |
| 批判反思者 | 中性分析体 | 质疑和反例本身构成文章结构 |
| 未来探索者 | 可启用编辑底色（评论体） | 叙事化推演，但保留分支条件 |

用户显式要求"亲切/编辑口吻/答复体"时，覆盖上述默认，对所有角色启用现代编辑底色。

## 三、模式×角色 的正交关系

深度（完整链路）和模式+角色是两个独立轴：
- **完整链路**：所有任务走同一完整链路 `crossframe → [sibling skills] → essay → review`
- **模式**：决定推断风险偏好（保守↔批判）
- **角色**：决定目标受众和分析立场（学术↔未来）

任何角色都可以和任何模式组合。例如：
- "保守 + 学术专家"：最稳的学术判断，多重边界
- "激进 + 未来探索者"：大胆推演 5-10 年演变
- "批判 + 批判反思者"：以反例、盲点和隐藏代价为组织方式的思想文章
- "客观 + 大众传播"：平衡的通俗解释

## 四、默认值

未指定时：
- 模式 = 🟢 客观
- 角色 = 🎓 学术专家

用户用"保守/客观/激进/批判"关键词切换模式，用"学术/实操/战略/通俗/批判反思/未来"或数字 1-6 切换角色。

## 五、与现有声口系统的关系

本选择器**替代**原有的 `editorial-base` 默认传递机制。声口不再由 suite 自动传给 essay，而是由角色自动决定。详见 `crossframe-suite/protocols/suite-dispatch-protocol.md` §4.1。
