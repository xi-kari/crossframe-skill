# Reader Reply Workflow Case

## 用户输入

有个读者问：“为什么我在关系里总是解释到很累，是不是我太敏感？”请像编辑回信一样回答。

## 调度提纲摘要

- 任务类型：关系困惑短答复。
- 工作流：`crossframe -> crossframe-dialogue -> review-lite`
- 必读 skill：结构诊断、短答复。
- 按需读取：亲密关系入口、解释劳动。
- 正文声口：编辑回信答复体，先接住提问者的疲惫。
- 不读取：`crossframe-essay`、`crossframe-public`、`crossframe-casebook`。
- 质量闸：轻量自检，重点检查是否把责任压回受伤者。

## 为什么这样排

用户要的是有洞察的回信，不是长文章。`crossframe-dialogue` 比 `crossframe-essay` 更合适；若用户要求扩成长文，再追加 `crossframe-essay`。
