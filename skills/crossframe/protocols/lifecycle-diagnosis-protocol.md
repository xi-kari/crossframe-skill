# 生命周期诊断协议

用于把 v2.0 的全周期演化过程转成可执行判断。阶段判断只能作为坐标，不能写成命运。

## 触发条件

- 用户问长期演化、组织周期、关系变化、制度生成或衰退。
- 输出需要判断“现在处在哪个阶段”。
- 现象同时包含成长、停滞、耗竭、修复、退出或重建。

## 执行顺序

1. 先完成 intake、五闸和机制候选。
2. 填写 `worksheets/lifecycle-stage-record.md`。
3. 区分主阶段、混合信号和阶段内位置。
4. 检查阶段判断是否会诱导过早行动、过度悲观或责任稀释。
5. 写出撤回条件、回退信号和升级信号。
6. 使用 `templates/lifecycle-output.md` 输出。

## 阶段坐标

| 阶段 | 人话识别 | 常见误判 |
| --- | --- | --- |
| 0 未成形 | 对象边界、共同指向和互动还不稳定 | 把愿望当系统 |
| 1 启动 | 有最小承诺、时间或资源进入 | 把口号当启动 |
| 2 扩展 | 行动被复制，承接需求上升 | 只看增长，不看承接 |
| 3 稳态 | 规则、角色、反馈形成常规 | 把稳定当健康 |
| 4 失衡 | 负荷集中、反馈失真、修复变慢 | 把耗竭归因给个体 |
| 5 重组 | 旧结构无法直接维持，需要重锚定或外部承接 | 把重组写成失败 |
| 6 退出/转移/再生 | 原系统目标切换、承接外移或新结构生成 | 把退出写成背叛或终结 |

## 输出边界

- 必须写“当前最像”而不是“已经注定”。
- 必须写混合阶段信号。
- 必须说明哪个证据会推翻阶段判断。
- 不得用阶段标签替代机制解释。
