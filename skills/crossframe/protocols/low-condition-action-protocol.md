# 低条件试探行动协议

## runtime hardening

本协议只负责专项推理步骤，不取消 `../SKILL.md` 的主运行链。凡本协议产生中心命题、机制候选、高风险概念判断、行动建议、公共定性、概率排序、退出/处置建议或可成文材料，必须先进入：

```text
v5-read-state-capsule
-> source-anchor-integrity-check
-> concept-fidelity-check
-> claim ledger
-> claim-ledger-check
```

没有 `claim_id` 的判断不得进入本协议输出。高风险概念未通过概念契约时，只能作为解释候选或表达转译，不能承担判断。
用于证据不足但问题紧急的情况。

目标不是用小证据做大判断，而是在低风险、可撤回、可观察的范围内创造下一轮证据。

## 使用条件

- 风险正在累积，不能无限等待。
- 当前证据不足以完整诊断。
- 行动可以低成本、低风险、可撤回。
- 行动结果能产生新的观察信号。

## 必须写清

- 目标：这个小动作要验证什么。
- 行动：具体做什么，不做什么。
- 风险边界：不会伤害谁，不会授权什么。
- 观察指标：出现什么信号算有效。
- 停止条件：什么情况必须停止。
- 升级条件：什么情况才进入更强行动或外部承接。

## 合格动作特征

- 小：不一次性摊牌、不永久改变关系或制度状态。
- 可撤回：行动后可以回退。
- 可观察：能产生明确反馈。
- 不转嫁：不把风险推给最弱的人。
- 不制造表演：不只是多开会、多写材料、多喊口号。

## 例句

> 现在不适合做强判断，但可以做一个低条件试探：把下一次反馈限定为一个具体事项，并要求它写回到角色、资源或时间表。如果只有口头安抚而没有任何写回，这就是重要信号。
