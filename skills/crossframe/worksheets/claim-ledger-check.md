# 命题台账检查表

本工作表用于检查 `claim-ledger.md` 是否足以支撑最终输出。它和 `source-anchor-integrity-check.md` 配套：源锚点检查回答“命题能否回指读源”，命题台账检查回答“正文是否只从已登记命题展开”。

默认不完整展示给普通用户；在结构洞察底稿中只摘要“命题台账状态 / 裸奔命题 / 降档决定”。

## 1. 台账存在性

| 项目 | 填写 |
| --- | --- |
| 是否存在 claim ledger | 是 / 否 |
| 是否覆盖中心命题 | 是 / 否 |
| 是否覆盖机制候选 | 是 / 否 |
| 是否覆盖高风险概念 | 是 / 否 / 未触发 |
| 是否覆盖行动建议 | 是 / 否 / 未触发 |
| 是否覆盖文章转译 | 是 / 否 / 未触发 |
| 是否覆盖公共事实 / 来源使用 | 是 / 否 / 未触发 |
| 是否写明撤回条件 | 是 / 否 |
| 是否写明发布边界 | 是 / 否 |

若台账缺失，不得进入文章正文、公共发布、强判断或 review 的 `substantive_pass`。

## 2. 命题强度检查

| 检查项 | 合格信号 | 失败信号 |
| --- | --- | --- |
| 中心命题 | 有 claim_id、事实锚点、判断档位、撤回条件 | 只有漂亮判断，没有来源或事实 |
| 机制候选 | 有 M1/M2，对应事实和反向条件 | 单机制解释一切 |
| 高风险概念 | 有概念卡、概念契约和 v5 连读包 | 术语直接当结论 |
| 行动建议 | 有行动上限和停止条件 | 直接建议公开、处置、退出、追责 |
| 来源使用 | 来源台账写清能证明什么和不能证明什么 | 来源气势抬高判断档位 |
| 写作技法 | 只绑定已有 claim_id | 技法制造新事实、新强判断或隐喻证明 |

## 3. 正文抽句反查

文章、评论、组织复盘、公共议题、案例、论辩、趋势推演和高责任主题，至少抽查 3-5 个正文短摘。

| 正文短摘 | 类型 | 对应 claim_id | 是否强于台账 | 处理 |
| --- | --- | --- | --- | --- |
|  | 中心命题 / 机制句 / 高风险概念 / 行动建议 / 概率排序 / 点睛句 | CL | 否 / 是 | 保留 / 删除 / 降档 / 标为表达转译 |

没有对应 `claim_id` 的正文短摘，不得用“文章表达需要”放行。

## 4. 判断档位上限

| 条件 | 最高允许档位 |
| --- | --- |
| 无 claim ledger | 轻量观察 |
| 有台账但缺撤回条件 | 轻量观察 |
| 证据闸 weak | 开放断言 |
| required_closure 未读完 | 开放断言 |
| source_anchor 缺失 | 本文推断 / 表达转译 |
| 概念契约 partial | 开放断言 |
| 概念契约 fail | 不得使用该概念承担判断 |
| 来源台账字段不完整 | 待核验分析 |
| 高责任主题且单一来源族 | internal-only / 待核验 |
| 七闸任一 blocked | blocked |
