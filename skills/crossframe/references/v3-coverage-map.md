# v3.0 覆盖地图

本文件用于回答“CrossFrame 相较 v3.0 是否还有遗漏”。它不替代原文，而是记录每个重要模块在 skill 中的承接位置。

章节级覆盖由三层文件共同维护：

- `v3-source-spine.md`：记录源章节节点、相邻关系、联读包和承接状态。
- `v3-section-digest-index.md`：记录逐节保真摘要、不可误读边界和相邻提醒。
- `continuity-bundles.md`：规定哪些板块不能单读概念卡。

## v3.0 新增重点

| 新增模块 | 承接文件 | 状态 | 使用边界 |
| --- | --- | --- | --- |
| 框架自诊、版本治理、根假设证伪、良性消亡 | `v3-term-fidelity.md`、`v3-change-rationale-from-patch.md`、`continuity-bundles.md` | 已联读约束/后台保真 | 框架也要接受外部审计，不能自证安全 |
| 共识程序、概念有效性分级、开放断言退场 | `procedural-judgment-responsibility.md`、`read-routing-map.md` | 已概念卡化/已联读约束 | 名义程序不等于有效程序，开放断言不得实际处置 |
| 可见性偏误、不透明区分、弱信号保护、AI 现实验证 | `visibility-opacity-weak-signals.md`、`malicious-compliance-ai-validation.md` | 已概念卡化 | AI 文本和漂亮格式不等于现实核验 |
| 无制度基础设施、无法退出、复杂创伤 | `no-institution-middle-path.md`、`trapped-subject-trauma-baseline.md` | 已概念卡化 | 不强行模拟制度，不要求不可退出者完成理想退出 |
| 爱的生成事件位置 | `love-generative-action.md`、`love-open-action.md` | 已概念卡化 | 爱既不能神化，也不能被塞进解释失败的剩余 |

| 使用门槛债、工具化、商业化、分裂协议 | `accessibility-toolization-split.md` | 已概念卡化 | 防止框架变成专家垄断、产品交付压力或解释权不平等 |
| 阶段 6、熵增、观测递归收束 | `observation-entropy-contraction.md`、`high-reflexivity-protocol.md` | 已概念卡化/已协议化 | 不能无限追踪，也不能用阶段/熵增包装不作为 |

## 判定结论

当前补全标准不是复制 v3.0 全文，而是让每个重要板块都有可追踪入口：普通使用走轻流程，高风险概念走概念卡和保真检查，连续板块走联读包和源结构连续性检查，深水区走专项 protocol 与 worksheet。
