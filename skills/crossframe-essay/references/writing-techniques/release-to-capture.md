# 欲擒故纵法

- 定义：为了最终抓住主题或对象，先放开、迂回、延宕，再收束。
- 适用文章类型：论辩/反驳文章、思想文章、案例叙事文章。
- 解决的问题：直线论证太硬，缺少张力。
- 操作步骤：先顺着一个表面解释走；逐步显出其不足；最后收回到真正判断。
- 原书操作要点（转述）：原书强调波澜必须来自认识、情感和事件本身的发展；起伏要有内在理由，不能为了热闹而制造曲折。本技法的关键不是记住名称，而是把“为了最终抓住主题或对象，先放开、迂回、延宕，再收束”落实到材料选择和段落位置上；执行时按“先顺着一个表面解释走；逐步显出其不足；最后收回到真正判断”推进。
- 段落动作：在成文时先标出它负责的段落位置，再用它解决“直线论证太硬，缺少张力”；该技法至少要完成一个可见动作，如立题、入场、串联、转折、反驳、回落或收束。
- CrossFrame 适配：只改变表达路径，不改变事实边界、判断档位、来源台账、连续联读包、读态胶囊和源锚点。若技法产生的新句子无法回指底稿，应降为表达转译或删除。
- 原书页码/OCR锚点：目录页码 p195；OCR 对应 PDF_PAGE 204 左右（标题 OCR 可能有少量错字）。本卡依据 OCR 研究笔记和对应页段转述，不复制原书长段。
- 好句类型：放开回收句：先暂时扩展，再把散开的材料收回中心。
- 段落前后关系：前接读者已经理解的表面解释；使用它制造合理起伏。后接揭示、降档、反向条件或责任链，不让波澜变成戏剧化断言。
- 文章类型微用法：评论和论辩中用它制造认知转折；趋势推演中用它展开分支；案例叙事中用它保留阅读张力；答复体只轻用，避免制造焦虑。
- 失败示例（转述）：失败时会为了起伏制造悬念、反转或戏剧性，把待核验内容写成事实。
- 文章类型用法：适用于论辩/反驳文章、思想文章、案例叙事文章。优先用于开篇抓手、转折、递进或中段反差，让文章有起落但不失焦。
- 失败形态：纵要服务擒，不能散漫失控；常见失败是为了好看而让技法接管命题、让比喻替代证据，或让段落动作越过胶囊边界。
- 搭配技法：以退为进法、曲径通幽法、故设悬念法。
- 误用风险：纵要服务擒，不能散漫失控。
- 输出自检：迂回后是否更准确地抓住问题？
