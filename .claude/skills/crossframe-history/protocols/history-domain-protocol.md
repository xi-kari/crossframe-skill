# History Domain Protocol

本协议用于把 CrossFrame v5 主干变量接入历史材料。它只负责领域接口层，不负责发布历史规律或修改 v5 主干。

## 1. 定义对象

先写清：

- 历史节点：事件、制度变化、长期结构或个案。
- 时间窗口：起点、关键节点、终点和观察余波。
- 空间范围：中央/地方、区域、边疆、跨帝国或全球链路。
- 制度对象：财政、军事、官僚、宗法、法律、户籍、劳役、市场、教育、公共卫生等。
- 主要主体：国家、地方精英、军队、官僚、宗族、商人、农民、女性、移民、奴役/劳役群体、殖民机构等。

同时写出本次历史输出档位：历史草稿档、历史接口分析档或正式历史分析档。档位由史料台账和材料距离决定，不由文章完整度、v5 包读取声明或质量闸声明决定。

## 2. 史料优先

历史事实优先来自历史材料。现代论文、专著、百科、课程和评论只允许承担三种功能：

1. 定位原始材料。
2. 提示争议或反方解释。
3. 帮助发现遗漏的材料族。

现代解读不得直接承担机制结论。若当前只有现代解读，输出必须降档为“待核验解释”。

若当前只有来源族或常识背景，没有具体史料、档案系列、法规/会议记录、数据库条目、版本或 source path，输出必须降为历史草稿档。

## 3. 主干变量限量

每次解释只选择 3-6 个 v5 主干变量。常见可选方向包括：

- 尺度窗口与尺度转移。
- 权力闭合、退出条件和成本转移。
- 反馈写回、责任链和授权链。
- 证据成本、低权力主体和弱信号保护。
- 治理连续性、多中心结构和制度惯性。
- 生命周期、递进闭环、观察反身性和高责任判断边界。

不得为了显得完整而堆概念。未读到 v5 源锚点时，不得把术语写成 v5 原义。

## 4. 历史因果降级

历史解释默认写成开放断言，不写成单因果定律。遇到以下情况必须降级：

- 关键史料只剩后世编纂。
- 时间链条中间缺口过大。
- 只能看到制度文本，看不到执行结果。
- 只能看到精英材料，看不到承受成本的主体。
- 多区域、多阶层或多制度同时变化，不能拆出主因。
- 反向案例足以说明机制不是必然。

## 5. 制度连续性/断裂翻译

写“连续”时必须说明连续的是：

- 名称连续。
- 法条/制度文本连续。
- 执行机构连续。
- 资源汲取方式连续。
- 责任链和成本承担方式连续。
- 主体经验或地方实践连续。

写“断裂”时也必须说明断裂层级。不得把王朝更替自动写成制度断裂，也不得把制度名称相同自动写成实际连续。

## 6. 王朝叙事误用防护

- “盛衰兴亡”只能作为叙事框架，不能直接替代机制分析。
- “王朝周期”不能写成结构必然。
- “统一/分裂”必须拆成财政、军事、交通、官僚、地方治理、合法性和外部压力等变量。
- 不能用现代价值判断替代史料边界。
- 不能把后世成败倒推成当时选择的唯一理性。

## 7. 解释质量登记

每次输出必须分三层：

- 解释顺畅：主干变量能低成本解释，且史料足以支撑。
- 解释高成本：主干变量能解释，但需要额外断代、区域、制度或材料补充。
- 解释不了：主干解释无法承担，需要登记失败类型。

失败类型固定为：史料不足、尺度错位、时间因果问题、领域概念缺失、框架误读。
