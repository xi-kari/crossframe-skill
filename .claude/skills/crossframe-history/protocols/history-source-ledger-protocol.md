# History Source Ledger Protocol

本协议规定历史材料的证据档位和台账写法。它补充 `../crossframe/references/source-ledger-workflow.md`，不替代十字段来源台账。

## 史料档位

| 档位 | 含义 | 可承担内容 |
| --- | --- | --- |
| H0 | 同时代原件、档案原页、出土材料、法律原文、奏议/诏令原件、可核验影像或数据库原始记录 | 事实字段、日期、主体、制度文本、原始行动记录 |
| H1 | 接近同时代材料、官方编年、正史本纪/志/表、实录、可靠再录原件、学术数据库逐页转录 | 事实框架、制度沿革、事件顺序；需标注编纂距离 |
| H2 | 后世编纂材料、类书、地方志、二手汇编、现代学术专著/论文对史料的引述 | 定位材料、争议提示、候选解释；不能单独承担强机制结论 |
| H3 | 百科、课程、评论、媒体文章、AI 生成摘要、未核验网页 | 线索和搜索入口；不得承担事实闭合或机制结论 |

## 台账字段

每个关键事实至少记录：

- source_id：
- 来源：
- 时间：
- 来源类型：
- 编纂距离/材料距离：
- 支持的 claim_id / 命题：
- 不能证明什么：
- 证据档位：
- 使用位置：
- 降档理由：
- 仍需补证处：

其中“不能证明什么”和“仍需补证处”不得省略。

历史史料台账必须能回指 `claim ledger`。如果关键历史事实只有 source_id、没有 claim_id，则只能作为材料背景，不能承担中心机制判断。

高风险概念必须读取 `../crossframe/references/concept-contracts/core-contracts.md` 中对应契约。未完成契约检查的历史概念只能作为解释候选，不得进入历史机制结论、适配器候选或公开表达边界。

## 来源粒度硬闸

来源族不是来源。以下写法只能作为线索，不能算史料台账完成：

- “官方文件”“公开史料”“现代学术研究”“回忆录”“口述史”“经济统计”“档案材料”。
- “已有历史研究显示”“学界一般认为”“公开材料可查”。
- 没有文件名、档案系列、法规名、会议记录名、数据库记录、页码/卷册/条目、版本或 source path 的泛称。

合格台账至少要把关键事实落到可核对象：

- 具体文献或材料名，例如某宪法文本、法令、会议记录、实录、档案系列、报纸期号、统计年鉴、数据库条目、考古报告、碑刻编号。
- 材料距离，例如同时代原件、官方再录、后世编纂、现代引用、待核 source path。
- 该材料能证明的最小命题，和它不能证明的命题。

如果当前只能写来源族，必须降为“历史草稿档”；如果有 source path 但未读原件，只能写“下一步可核验性提升”，不能写事实闭合。

## 输出档位判定

| 输出档位 | 台账要求 | 评审上限 |
| --- | --- | --- |
| 历史草稿档 | 无具体史料台账，或只有来源族/常识背景/现代概述 | 不得高于 `draft_pass` |
| 历史接口分析档 | 至少对核心事实列出具体来源或 source path、证据档位、不能证明什么和仍需补证处 | 可给 `structural_pass`，不得自动给正式闭合 |
| 正式历史分析档 | 关键事实有具体 H0/H1 或可核验 H2 台账，多源互证或说明单源足够 | 才可考虑 `substantive_pass` |

“读了 CrossFrame 包”“写了 V5-H 锚点”“文章成文完整”不提升历史输出档位。历史档位只由史料台账、材料距离和可撤回条件决定。

## 编纂距离标注

常用历史材料必须标注距离和风险：

- 正史：可作为历史材料，但需标注修史朝代、政治立场和材料来源风险。
- 编年体：可帮助时间线，但不得自动等同原始见证。
- 实录：接近制度运行，但仍有删改、修辞和政治筛选风险。
- 地方志：可提供地方制度/人物/灾害/赋役材料，但需注意修志年代和地方精英视角。
- 出土材料/档案原件：优先级高，但仍要检查断片、释读、编号、上下文和代表性。

## 不能越界

- 只有制度文本时，不得直接推出制度实际执行效果。
- 只有惩罚记录时，不得直接推出全部社会经验。
- 只有精英文本时，不得直接代替底层主体经验。
- 只有 source path 或 finding aid 时，不得写成已经读到原件。
- 只有现代解释时，不得写成史料已经闭合。
