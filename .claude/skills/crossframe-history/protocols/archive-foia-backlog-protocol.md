# Archive / FOIA Backlog Protocol

本协议用于处理档案访问、FOIA、finding aid、目录路径和个案事实闭合。它的目标是提升下一步可核验性，而不是用目录替代事实。

## 状态区分

| 状态 | 含义 | 允许声明 | 禁止声明 |
| --- | --- | --- | --- |
| source path | 找到档案馆、系列、卷宗、目录或数据库路径 | 可以申请/调阅的方向更明确 | 已经看到原件、事实已闭合 |
| finding aid | 找到馆藏说明、箱号、系列描述或访问限制 | 可能存在相关材料 | 材料内容已经支持某结论 |
| H0/H1 extracted | 已看到原页、影像、官方记录或可靠逐页转录，并抽取字段 | 对字段内命题可局部闭合 | 超出字段范围的生命结果/机制结论 |
| public closeout backlog | 需要机构最终信、结案、执行或监测材料 | 公共程序闭合仍待核验 | 机构承诺已经兑现 |
| FOIA pending | 申请尚未提交或尚未收到答复 | 仍是待证路径 | 事实已经升级 |

## 申请包最小字段

每个 backlog 个案都必须写：

- 案件：
- 当前证据档位：
- 已有 source path：
- 缺失原件：
- 待请求机构/系列/卷宗：
- 目标字段：
- 能闭合的命题：
- 不能闭合的命题：
- 收到材料后的升级条件：
- 收到材料后的降级/撤回条件：
- 申请状态：

## 升级规则

只有满足以下条件，才允许从 backlog 升级为局部事实闭合：

1. 取得 H0/H1 原件、影像、官方记录或可靠逐页转录。
2. 抽取了主体、日期、地点、制度动作、结果字段或其他目标字段。
3. 写明字段能支持的 claim_id / 命题和不能支持的 claim_id / 命题。
4. 保留材料编号、页码、卷宗号或 URL。
5. 对照至少一个反向条件。

## 降级规则

遇到以下情况必须降级：

- 目录存在但材料未开放。
- 申请范围过宽，无法定位具体记录。
- 只收到机构概括回复，没有原始字段。
- 原件只证明制度文本，不证明执行结果。
- 原件只证明个案动作，不证明群体机制。
- 同案号 closeout 缺失，只有 opening letter、agreement 或 public statement。

## 输出边界

archive-access / FOIA backlog 输出必须使用“待证”“申请路径”“下一步核验”这类语言。不得把 backlog 写成“已证明”“已闭合”“已经还原”。
