# 尺度转移闸

尺度转移是 CrossFrame 的关键能力，也是最容易误用的地方。

## 允许的尺度转移

- 从个体痛苦上升到关系模式，以看见反复发生的机制。
- 从团队症状上升到组织流程，以看见反馈为何无法写回。
- 从制度个案上升到权力结构，以看见程序为何失效。
- 从宏观叙事下降到具体接口，以找出责任链和修复点。

## 不允许的尺度转移

- 用宏大历史解释让具体受害者闭嘴。
- 用“大家都是时代产物”取消具体责任。
- 用“系统问题”掩盖某些人拥有改变条件的能力。
- 用个体情绪替代制度证据。
- 用道德标签替代机制分析。

## 检查问题

1. 本次转移解释了什么新增事实？
2. 它是否保留了原尺度的痛苦与责任？
3. 它是否让行动边界更清楚？
4. 它是否让弱者承担更多证明成本？
5. 它是否让强者更容易逃避责任？

## 输出原则

可以说：

> 从关系尺度看，这是回流断裂；从个体尺度看，它真实表现为长期疲惫和自我怀疑。后者不能被前者取消。

不要说：

> 这只是结构问题，所以你不该痛苦。
