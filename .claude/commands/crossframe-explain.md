# /crossframe-explain

使用 CrossFrame 解释一个概念、思想、现象或术语边界。适合“解释虚无主义”“某概念怎么看”“这个说法在 CrossFrame 里是什么意思”。

执行要求：

1. 读取 `skills/crossframe/SKILL.md`。
2. 读取 `skills/crossframe/protocols/concept-explanation-protocol.md`。
3. 读取 `skills/crossframe/references/read-routing-map.md` 与 `skills/crossframe/references/v5-term-fidelity.md`。
4. 如果涉及高风险概念，读取对应 `skills/crossframe/references/concept-cards/`。
5. 概念解释需要深度或涉及 v5.0 高风险概念时，读取 `skills/crossframe/references/integrity-check.md`，需要展开源结构时再按需读取 `skills/crossframe/references/continuity-bundles.md` 与 `skills/crossframe/references/v5-section-digest-index.md`。
6. 输出顺序：推理提纲 -> 人话解释 -> CrossFrame 映射 -> 误读边界 -> 可观察现实信号 -> 可展开诊断入口。
7. 不要写成百科解释，也不要把概念直接用于人格诊断。
8. 对第一因、生命是什么、虚无主义、存在意义等哲学/意义类问题，先做尺度拆分并给结构性开放断言；只有完全无法结构化时才退回框架边界。
