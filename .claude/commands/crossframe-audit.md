# /crossframe-audit

使用 CrossFrame 做高责任、反俘获、强判断或 v5.0 覆盖审计。适合处分、名誉、权利、资源、公共记忆、AI 合规材料、框架边界和“有没有概念缺失”的问题。

执行要求：

1. 读取 `skills/crossframe/SKILL.md`。
2. 读取 `skills/crossframe/references/read-routing-map.md`。
3. 必读：`skills/crossframe/protocols/anti-capture-protocol.md`、`skills/crossframe/protocols/proposition-verification-protocol.md`、`skills/crossframe/protocols/framework-boundary-protocol.md`。
4. 必读 `skills/crossframe/references/runtime-read-policy.md`、`skills/crossframe/references/read-routing-map.md` 和 `skills/crossframe/references/continuity-closure-map.md`；需要展开审计时，再用 `skills/crossframe/references/continuity-bundles.md` 和 `skills/crossframe/worksheets/source-continuity-check.md` 检查是否只读了孤立概念。
5. 默认按 v5.0 完整度读取 `skills/crossframe/references/v5-coverage-map.md`、`skills/crossframe/references/v5-source-spine.md` 和 `skills/crossframe/references/v5-section-digest-index.md`；只有用户明确问 v3.0 历史版本时，才读取 v3 文件做历史对照。
6. 先输出审计提纲：对象、事实边界、证据成本、权力密度、申诉/复核、判断档位、连续联读包、保真检查、下一步。
7. 不得把开放断言当作高责任处置依据；不得把 AI 生成报告当独立强证据。
