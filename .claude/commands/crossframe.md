# /crossframe

使用 CrossFrame 对用户给出的对象做跨尺度结构诊断、推演、开放断言、强判断验证或深水区审查。

如果用户目标是写中文文章、长文、评论、思想文章或批判性洞察文章，请改用 `/crossframe-essay`，不要只按诊断模板输出。

执行要求：

1. 读取 `skills/crossframe/SKILL.md`。
2. 读取 `skills/crossframe/references/read-routing-map.md`，判断任务类型：诊断、推演、开放断言、强判断验证、高反身性、亲密关系、疗愈转移、公共制度、低条件行动、高责任审查、框架边界、生命周期、递进闭环、势场解离、治理连续性、超大规模压力测试、表达翻译、理论后台或概念解释。
3. 如果判断依赖高风险概念，读取对应概念卡和 `skills/crossframe/references/v3-term-fidelity.md`。
4. 高责任、公共制度、亲密关系、长期演化、深度分析和文章输出，优先读取 `skills/crossframe/references/integrity-check.md`；需要审计时再按需读取 `skills/crossframe/references/continuity-bundles.md`、`skills/crossframe/references/v3-source-spine.md`、`skills/crossframe/references/v3-section-digest-index.md`，并用 `skills/crossframe/worksheets/source-continuity-check.md` 检查是否读少。
5. 先给简短推理提纲：诊断对象、事实边界、尺度窗口、机制候选、判断档位、本次读取的概念、本次连续联读包、本次读取路径、本次保真/边界检查、下一步。
6. 再输出面向用户的人话判断。
7. 若证据不足或必须联读但未联读，降低判断档位，不要伪装成强判断。

用户输入：

`$ARGUMENTS`
